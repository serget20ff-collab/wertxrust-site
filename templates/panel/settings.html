Я делаю проект WertxRust: сайт на Django и отдельный Rust-сервер. Мне нужен локальный C# серверный агент/скрипт для Rust, который будет взаимодействовать с Django-сайтом.

Архитектура сайта:
- Главный идентификатор игрока: SteamID64.
- Django хранит SteamProfile.
- У SteamProfile есть:
  - steam_id
  - nickname
  - last_known_ip — IP входа на сайт
  - last_server_ip — последний IP входа на Rust-сервер
  - last_seen_on_server_at — последний вход на Rust-сервер
  - project_role
  - balance_rub
- Для серверных входов есть модель ServerPlayerConnection:
  - profile
  - server
  - server_ip
  - nickname_at_moment
  - first_seen_at
  - last_seen_at
  - connections_count
  - raw_payload
- На сайте есть ServerApiToken:
  - name
  - server
  - token
  - is_active
  - allowed_ip
  - last_used_at

Мне нужен C# агент, который:
1. Работает рядом с Rust-сервером.
2. Отслеживает вход игрока.
3. Получает SteamID64, ник игрока, IP игрока на сервере, ID/название сервера.
4. Отправляет POST-запрос на Django API с API-токеном.
5. Django должен обновить или создать SteamProfile, создать/обновить ServerPlayerConnection и обновить last_server_ip у игрока.
6. В будущем агент должен принимать команды с сайта: муты, варны, проверки, выдача покупок, привилегии и киты.
7. Нужна безопасная структура: API-токен в заголовке, проверка IP сервера, логирование ошибок, повторная отправка при сбоях.

Помоги спроектировать C# агент и Django API endpoints для этого.
